# دليل تشغيل منصة التعلم الإلكتروني (LMS)

هذا الدليل مخصص للمطورين الجدد الذين يرغبون في تشغيل المشروع على أجهزتهم المحلية وفهم هيكليته بالكامل.

## 🛠️ التقنيات والمكتبات المستخدمة

يعتمد المشروع على أحدث تقنيات تطوير الويب:

- **الواجهة الخلفية (Backend):**
  - **Laravel 12 / 11** (إطار العمل الأساسي ببيئة PHP 8.2).
  - **MySQL** (قاعدة البيانات الأساسية).
  - **Spatie Permission** (`spatie/laravel-permission`): لإدارة الأدوار (Roles) والصلاحيات (Permissions).
  - **Spatie Media Library** (`spatie/laravel-medialibrary`): لإدارة رفع وحفظ الملفات (صور البطاقات، الصور الشخصية للمدربين، غلاف الكورسات، الفيديوهات).
  - **Laravel Reverb**: لبث الأحداث (Broadcasting) ودعم الإشعارات اللحظية.

- **الواجهة الأمامية (Frontend):**
  - **Inertia.js**: لربط Laravel بـ React دون الحاجة لبناء API منفصلة.
  - **React.js + TypeScript**: لبناء واجهات مستخدم تفاعلية آمنة الأنواع (Type-safe).
  - **Tailwind CSS**: لتنسيق الواجهات بتصميم عصري ومتجاوب.

- **ميزات إضافية:**
  - **Gemini AI**: للدردشة الذكية (Chatbot) مع الطلاب باستخدام مفتاح API.

## 🚀 خطوات التشغيل خطوة بخطوة (باستخدام MySQL)

### 1. تثبيت المتطلبات الأساسية
تأكد من توفر البرامج التالية على جهازك:
- **PHP** (الإصدار 8.2 أو أحدث).
- **Composer** (لإدارة حزم PHP).
- **Node.js & npm** (لإدارة حزم الواجهة الأمامية).
- **XAMPP** أو أي خادم محلي لتشغيل **MySQL**.

### 2. نسخ المشروع وتثبيت الحزم
افتح موجّه الأوامر (Terminal) في مجلد المشروع، ونفذ الأوامر التالية:

```bash
# 1. تثبيت حزم Laravel (Backend)
composer install

# 2. تثبيت حزم React (Frontend)
npm install
```

### 3. إعداد البيئة (Environment)
قم بنسخ ملف البيئة الافتراضي وتسميته بـ `.env`:

```bash
cp .env.example .env
```
افتح ملف `.env` وقم بتعديل بيانات الاتصال بقاعدة البيانات لتتطابق مع خادم MySQL لديك:
```env
DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=lms
DB_USERNAME=root
DB_PASSWORD=
```
*(ملاحظة: تأكد من إنشاء قاعدة بيانات فارغة باسم `lms` في MySQL قبل الخطوة التالية)*.

### 4. توليد مفتاح التطبيق وربط التخزين
```bash
php artisan key:generate
php artisan storage:link
```
*(ربط التخزين ضروري جداً لكي تظهر الصور ومقاطع الفيديو التي يتم رفعها).*

### 5. تجهيز قاعدة البيانات والبيانات الافتراضية (Seeding)
يحتوي النظام على بيانات افتراضية (Seeders) تقوم بإنشاء الأدوار، المستخدمين، و 10 دورات تجريبية مع دروسها واختباراتها:

```bash
# قم بإنشاء الجداول وإدخال البيانات الافتراضية دفعة واحدة
php artisan migrate:fresh --seed
```

### 6. تشغيل الخوادم
لفتح المشروع، ستحتاج لفتح **نافذتين (Terminal)**:

**النافذة الأولى (لشتغيل خادم PHP):**
```bash
php artisan serve
```

**النافذة الثانية (لتشغيل خادم React+Vite):**
```bash
npm run dev
```

الآن يمكنك فتح المتصفح على الرابط: `http://localhost:8000`

## 🔑 حسابات الدخول الافتراضية

بعد تشغيل الـ `Seeders`، ستتوفر الحسابات التالية لتجربة كافة صلاحيات النظام. **كلمة المرور لجميع الحسابات هي: `password`**

| الدور (Role) | البريد الإلكتروني | كلمة المرور | ميزات الحساب |
|--------------|-------------------|------------|--------------|
| **مدير النظام (Admin)** | `admin@lms.com` | `password` | قبول/رفض المدربين، إدارة الأقسام، صلاحيات كاملة. |
| **مدرب (Instructor)** | `instructor@lms.com` | `password` | إنشاء دورات، إضافة دروس واختبارات، طلب سحب أرباح. |
| **طالب (Student)** | `student@lms.com` | `password` | تصفح الدورات، التسجيل فيها، حل الاختبارات، استخدام الذكاء الاصطناعي. |

## 📁 هيكلية الملفات الأساسية للمشروع

لفهم أين تكتب الأكواد، هذه هي الأماكن التي ستحتاجها:

### 1. مجلدات الـ Backend (Laravel - PHP)
- **`app/Models/`**: يحتوي على الجداول وعلاقاتها (User, Course, Section, Lesson, Quiz ...).
- **`app/Http/Controllers/`**: يحتوي على المنطق البرمجي (الـ Controllers)، وهو مقسم حسب الأدوار:
  - `Admin/`: لوحة تحكم الإدارة.
  - `Instructor/`: لوحة المدرب (الكورسات والاختبارات).
  - `Student/`: تصفح الكورسات ومشغل الفيديو (`CourseLearnController.php`).
- **`routes/web.php`**: ملف توجيه الروابط (Routes)، مقسم باستخدام الـ `Middleware` لحماية المسارات حسب دور المستخدم (`role:Student`, `role:Instructor`).
- **`database/migrations/`**: ملفات بناء جداول قاعدة البيانات.
- **`database/seeders/`**: ملفات حقن البيانات الافتراضية المذكورة أعلاه.

### 2. مجلدات الـ Frontend (React - TypeScript)
جميع ملفات الواجهة موجودة في مجلد **`resources/js/`**:
- **`resources/js/pages/`**: صفحات النظام مقسمة حسب الأدوار:
  - `admin/`: واجهات قبول المدربين وغيرها.
  - `instructor/`: واجهات بناء الكورس والاختبارات ومتابعة طلب التقديم.
  - `student/`: واجهات تصفح الكورس، مشغل الفيديو (`CoursePlayer.tsx`)، والشات بوت.
- **`resources/js/components/`**: المكونات القابلة لإعادة الاستخدام (مثل `AIChatbot.tsx`، الأزرار، وحقول الإدخال).
- **`resources/js/layouts/`**: قوالب الصفحات الأساسية التي تحتوي على الشريط العلوي (Navbar) والقائمة الجانبية (Sidebar).
- **`resources/js/types/index.ts`**: ملف هام جداً يحتوي على تعريف أنواع البيانات (TypeScript Interfaces) لضمان أمان البيانات بين الـ Backend والـ Frontend.

## 🤖 إضافة مفتاح الذكاء الاصطناعي (Gemini)
إذا أردت تفعيل الشات بوت الخاص بالطلاب، افتح ملف `.env` وأضف المتغير التالي:
```env
GEMINI_API_KEY=YOUR_API_KEY_HERE
```
*(استبدل `YOUR_API_KEY_HERE` بمفتاح API الخاص بك من Google AI Studio).*
